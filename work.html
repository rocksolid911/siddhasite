Title:

Covid-19-Data-Pipeline-Based-On-Messaging-and-Analysis | Data Visualization

Description:

The purpose of this project is to have a look at the current state of COVID-19 using visualizations from different geographical perspectives. The plots have been created by using data visualization tools such as Tableau.

  • Data Extraction: The purpose is to collect the real time streaming data from COVID19 open API for every 5 minutes into the ecosystem using NiFi and to process it and store it in the data lake on AWS.
  • Data Processing: Data processing includes parsing the data from complex JSON format to csv format then publishing to Kafka for persistent delivery of messages into PySpark for further processing.The processed data is then fed into output Kafka topic which is inturn consumed by Nifi and stored in HDFS.
  • Data Visualization: In the following section, we look at the disease from the World Health Organization's regional perspective. We will compare the different regions based on their total cases, total deaths using Tableau.

Technologies:

Hadoop, Apache PySpark, Apache-hive, Nifi, Kafka, Zookeeper, Airflow, Tableau

Title:

An Examination of Fatal Force by Police in the US | Classification & Multi Classification Problem

Description:

Title:

Predicting number of Covid19 deaths using Time Series Analysis (ARIMA MODEL) | Time Series Forecasting Problem

Description:

In this project, a Time Series Analysis (ARIMA Model) predicts the number of deaths in the United States starting from August 1st – August 21st and August 1st – November 1st. The goal of this project was to compare our results with the projections that have been made by CNN. In addition to the comparison to CNN the aim was to get a deeper understanding of Time Series Analysis and the ARIMA model as well.

Technologies:

R, Ggplot2, Tidyverse, Zoo, aTSA, Ggplot2tseries, Forecast, Lubridate.
